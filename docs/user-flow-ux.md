# FanoosAI - فلو کاربری و صفحات مورد نیاز UI/UX

## ۱. احراز هویت (Authentication) — فقط با شماره تلفن (SMS OTP)

> روش ایمیل/رمز عبور **غیرفعال** است. تنها روش ثبت‌نام و ورود از طریق OTP پیامکی می‌باشد.

### ۱-۱. صفحه ورود شماره تلفن (مرحله اول)
| بخش | توضیحات |
|---|---|
| **ورودی‌ها** | شماره تلفن همراه |
| **تشخیص کاربر جدید/قدیمی** | backend تشخیص می‌دهد کاربر جدید است یا قبلاً ثبت‌نام کرده |
| **کاربر جدید** | پس از وارد کردن شماره، به صفحه تکمیل اطلاعات هدایت شود |
| **کاربر قدیمی** | دکمه «ارسال کد» نمایش داده شود |
| **API** | `POST /auth/otp/sms/register` (کاربر جدید) یا `POST /auth/otp/sms/request` (کاربر قدیمی) |

### ۱-۲. صفحه تکمیل اطلاعات (فقط کاربر جدید)
| بخش | توضیحات |
|---|---|
| **شرط** | فقط برای کاربرانی که اولین بار است ثبت‌نام می‌کنند |
| **ورودی‌ها** | نام، نام‌خانوادگی، ایمیل (اختیاری) |
| **دکمه** | دریافت کد تأیید |
| **API** | `POST /auth/otp/sms/register` (همراه با فیلدهای name) |

### ۱-۳. صفحه ورود کد تأیید (مرحله دوم)
| بخش | توضیحات |
|---|---|
| **ورودی‌ها** | کد ۶ رقمی دریافتی از SMS |
| **وضعیت‌ها** | تایمر ۱۲۰ ثانیه‌ای برای انقضای کد |
| **دکمه ارسال مجدد** | بعد از ۶۰ ثانیه فعال می‌شود |
| **محدودیت** | حداکثر ۵ تلاش ناموفق |
| **خطا** | کد نامعتبر، کد منقضی شده، تلاش‌های زیاد |
| **موفق** | دریافت JWT و هدایت به داشبورد پروژه‌ها |
| **API** | `POST /auth/otp/sms/verify` |

## ۲. پروژه‌ها (Projects)

### ۲-۱. داشبورد / لیست پروژه‌ها
| بخش | توضیحات |
|---|---|
| **هدف** | نمایش لیست پروژه‌های کاربر با قابلیت جستجو |
| **کارت هر پروژه** | نام، توضیحات (خلاصه)، تاریخ ایجاد، تعداد پرامپت‌ها، تعداد اجراها |
| **دکمه‌ها** | ساخت پروژه جدید، کلیک روی پروژه (ورود به جزئیات) |
| **حالت خالی** | "هنوز پروژه‌ای نساخته‌اید" + دکمه ساخت پروژه |
| **API** | `GET /projects/` |

### ۲-۲. صفحه ساخت پروژه جدید
| بخش | توضیحات |
|---|---|
| **ورودی‌ها** | نام پروژه (الزامی)، توضیحات (اختیاری)، آدرس وبسایت (اختیاری) |
| **دکمه‌ها** | ایجاد پروژه، انصراف |
| **اعتبارسنجی** | نام: ۱-۲۰۰ کاراکتر، توضیحات: max ۱۰۰۰ کاراکتر |
| **API** | `POST /projects/` |

### ۲-۳. صفحه جزئیات پروژه
| بخش | توضیحات |
|---|---|
| **هدر** | نام پروژه، توضیحات، وبسایت، دکمه ویرایش، دکمه حذف |
| **داشبورد تحلیلی پروژه** | |
| - کارت آمار | تعداد پرامپت‌ها، تعداد مدل‌های فعال، تعداد اجراها، تعداد برندهای کشف‌شده |
| - کارت وضعیت | آخرین اجرای موفق، تعداد اجراهای موفق/ناموفق |
| **لیست پرامپت‌ها** | جدول/لیست پرامپت‌ها با جزئیات (متن، مدل‌ها، آخرین اجرا، وضعیت) |
| **تاریخچه اجراها** | جدول paginated از تمام اجراهای پروژه |
| **دکمه‌ها** | ساخت پرامپت جدید، مشاهده رتبه‌بندی برندها |
| **API** | `GET /projects/{id}`, `GET /projects/{id}/dashboard`, `GET /projects/{id}/history`, `GET /projects/{id}/runs` |

### ۲-۴. صفحه ویرایش پروژه
| بخش | توضیحات |
|---|---|
| **ورودی‌ها** | همان فیلدهای ساخت (از قبل پر شده) |
| **دکمه‌ها** | ذخیره تغییرات، انصراف |
| **API** | `PUT /projects/{id}` |

## ۳. پرامپت‌ها (Prompts)

### ۳-۱. صفحه ساخت پرامپت جدید
| بخش | توضیحات |
|---|---|
| **ورودی متنی** | Textarea برای نوشتن پرامپت (max ۱۰۰۰۰ کاراکتر) |
| **انتخاب مدل‌های AI** | لیست چک‌باکسی از مدل‌های فعال (نام مدل + provider) |
| **پیش‌نمایش** | نمایش تعداد کاراکترها، مدل‌های انتخاب‌شده |
| **دکمه‌ها** | ایجاد پرامپت، انصراف |
| **API** | `GET /ai-models` (برای بارگذاری لیست مدل‌ها), `POST /projects/{id}/prompts` |

### ۳-۲. صفحه جزئیات پرامپت
| بخش | توضیحات |
|---|---|
| **متن پرامپت** | نمایش کامل متن (قابل کپی) |
| **مدل‌های متصل** | لیست مدل‌های AI متصل به این پرامپت |
| **دکمه‌ها** | اجرا (Run)، حذف (آرشیو)، افزودن/حذف مدل |
| **بخش نتایج آخرین اجرا** | جدول نتایج آخرین run |
| **API** | `GET /projects/{id}/prompts/{pid}`, `DELETE /projects/{id}/prompts/{pid}` |

### ۳-۳. صفحه اجرای پرامپت (Run)
| بخش | توضیحات |
|---|---|
| **وضعیت** | صفحه لودینگ / نمایش پیشرفت اجرا روی هر مدل |
| **نتایج** | جدول خلاصه: وضعیت هر run، تعداد برندهای پیدا شده، برندهای جدید، برندهای تکراری |
| **خطا** | نمایش پیام خطا در صورت شکست |
| **API** | `POST /projects/{id}/prompts/{pid}/run` |

## ۴. مدل‌های AI (AI Models)

### ۴-۱. صفحه لیست مدل‌های AI
| بخش | توضیحات |
|---|---|
| **جدول مدل‌ها** | نام مدل، provider، وضعیت فعال/غیرفعال |
| **دکمه** | همگام‌سازی مدل‌ها از Gateway |
| **API** | `GET /ai-models` , `POST /ai-models/sync` |

## ۵. تحلیل‌ها و رتبه‌بندی (Analytics)

### ۵-۱. صفحه رتبه‌بندی برندها (Rankings)
| بخش | توضیحات |
|---|---|
| **هدف** | نمایش رتبه‌بندی برندهای استخراج‌شده از یک پرامپت |
| **فیلترها** | انتخاب پرامپت، انتخاب مدل AI، انتخاب برند |
| **جدول** | نام برند، دامنه، رتبه، مدل AI، تاریخ |
| **صفحه‌بندی** | paginated |
| **API** | `GET /prompts/{id}/latest-rankings`, `GET /prompts/{id}/rankings` |

### ۵-۲. صفحه جزئیات برند
| بخش | توضیحات |
|---|---|
| **هدر** | نام برند، دامنه |
| **آمار** | تعداد کل ظهورها، میانگین رتبه، بهترین/بدترین رتبه، اولین/آخرین مشاهده |
| **تاریخچه** | روند رتبه برند در طول زمان |
| **API** | `GET /brands/{id}`, `GET /brands/{id}/history` |

### ۵-۳. صفحه تاریخچه اجراها
| بخش | توضیحات |
|---|---|
| **جدول** | تاریخ، پرامپت، مدل AI، وضعیت، تعداد برندها |
| **فیلترها** | محدوده تاریخ، پرامپت، مدل AI |
| **صفحه‌بندی** | paginated |
| **API** | `GET /projects/{id}/history`, `GET /prompts/{id}/history` |

## نکات UI/UX
- همه صفحات باید **راست‌چین (RTL)** باشند
- از **حالت خالی (Empty State)** برای هر لیست استفاده شود
- برای پرامپت‌ها از **Syntax highlighting** با فونت monospace استفاده شود
- اجرای پرامپت ممکن است زمان‌بر باشد → **لودینگ با درصد پیشرفت** نمایش داده شود
- جدول رتبه‌بندی برندها قابلیت **مرتب‌سازی و فیلتر** داشته باشد
- تمام APIهای نیازمند احراز هویت، اگر توکن منقضی شده باشد → کاربر به صفحه لاگین هدایت شود
