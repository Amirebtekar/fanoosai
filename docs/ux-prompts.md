# پرامپت‌های طراحی UI/UX — FanoosAI

> هر پرامپت مختص یک صفحه است. به ترتیب اولویت طراحی، تیم UI/UX می‌تواند جلو برود.

---

## پرامپت ۱: صفحه داشبورد (لیست پروژه‌ها)

### هدف صفحه
صفحه اصلی بعد از ورود کاربر. لیست پروژه‌های کاربر رو نشون میده.

### API مصرفی
- `GET /projects/` (لیست پروژه‌های کاربر)

### داده‌های هر پروژه
- `name` — نام پروژه
- `description` — توضیحات
- `created_at` — تاریخ ساخت

### المان‌های مورد نیاز
- **Header**: لوگو + دکمه «پروژه جدید» + نام کاربری و دکمه خروج
- **لیست کارتی**: هر کارت شامل نام، توضیحات (محدود به ۲ خط)، تاریخ نسبی (مثلاً «۳ روز پیش»)
- **کلیک روی کارت**: ورود به صفحه جزئیات پروژه
- **جستجو**: Search bar برای فیلتر بر اساس نام پروژه

### وضعیت‌ها
- **لودینگ**: Skeleton loader به شکل کارت‌های خالی
- **خالی**: تصویر иллюстрация + متن «هنوز پروژه‌ای نداری. اولین پروژه‌ات رو بساز!» + دکمه «ساخت پروژه»
- **خطا**: پیام خطا با دکمه «تلاش مجدد»

### نکات UI/UX
- راست‌چین (RTL)
- دسکتاپ: ۳ ستون / تبلت: ۲ ستون / موبایل: ۱ ستون
- حالت Hover روی کارت: سایه بیشتر + pointer cursor
- آیکون حذف در گوشه کارت (با دیالوگ تأیید)

## پرامپت ۲: صفحه ساخت پروژه جدید

### هدف صفحه
فرم ایجاد پروژه جدید.

### API مصرفی
- `POST /projects/`

### فیلدهای فرم
| فیلد | نوع | الزامی | محدودیت |
|------|------|--------|----------|
| نام پروژه | Text input | بله | ۱-۲۰۰ کاراکتر |
| توضیحات | Textarea | خیر | max ۱۰۰۰ کاراکتر |
| آدرس وبسایت | URL input | خیر | max ۵۰۰ کاراکتر |

### المان‌های مورد نیاز
- **فرم** با سه فیلد بالا
- **دکمه «ایجاد پروژه»** — غیرفعال تا وقتی فرم معتبر نباشه
- **دکمه «انصراف»** — برگشت به داشبورد
- **اعتبارسنجی لحظه‌ای** — مثلاً "نام پروژه الزامی است" زیر فیلد خالی

### وضعیت‌ها
- **ارسال**: دکمه به حالت لودینگ + غیرفعال شدن فرم
- **خطای سرور**: نمایش پیام خطا بالای فرم (مثلاً "مشکلی پیش آمد، دوباره تلاش کن")
- **موفق**: هدایت خودکار به صفحه جزئیات پروژه

### نکات UI/UX
- فرم تمیز و مینیمال باشه
- placeholderهای راهنما داخل هر فیلد
- responsive باشه

## پرامپت ۳: صفحه جزئیات پروژه

### هدف صفحه
مشاهده جزئیات کامل یک پروژه + آمار تحلیلی + لیست پرامپت‌ها.

### APIهای مصرفی
- `GET /projects/{id}` — اطلاعات پروژه
- `GET /projects/{id}/dashboard` — آمار تحلیلی
- `GET /projects/{id}/prompts` — لیست پرامپت‌های پروژه
- `GET /projects/{id}/runs` — آخرین اجراها (paginated)

### ساختار صفحه

**۱. هدر پروژه:**
- نام پروژه (عنوان بزرگ)
- توضیحات
- وبسایت (لینک کلیکable)
- دکمه‌های «ویرایش», «حذف»

**۲. کارت‌های آمار (Dashboard Summary):**
داده از `GET /projects/{id}/dashboard`:
- `prompt_count` — تعداد پرامپت‌ها
- `active_model_count` — تعداد مدل‌های فعال
- `run_count` — تعداد کل اجراها
- `brand_count` — تعداد برندهای کشف‌شده
- `successful_run_count` — اجراهای موفق
- `failed_run_count` — اجراهای ناموفق
- `last_successful_run` — آخرین اجرای موفق (تاریخ نسبی)

پیشنهاد: ۴ کارت در یک ردیف (پرامپت‌ها - مدل‌ها - اجراها - برندها)

**۳. لیست پرامپت‌ها (بخش اصلی):**
- جدول یا لیست با ستون‌های: متن پرامپت (خلاصه), مدل‌های متصل, آخرین اجرا, وضعیت, تعداد برندها
- دکمه «پرامپت جدید»
- کلیک روی هر ردیف → ورود به صفحه جزئیات پرامپت

**۴. تاریخچه آخرین اجراها:**
- جدول paginated آخرین runها
- ستون‌ها: تاریخ, پرامپت, مدل AI, وضعیت (موفق/ناموفق), تعداد برندها

### وضعیت‌ها
- **لودینگ**: Skeleton برای کارت‌ها و جداول
- **خالی (بدون پرامپت)**: «هنوز پرامپتی اضافه نشده» + دکمه ساخت پرامپت
- **خطا**: پیام خطا + دکمه تلاش مجدد

### نکات UI/UX
- تب‌بندی بین بخش‌های مختلف (پرامپت‌ها, اجراها, برندها) — یا اسکرول عمودی
- نمودار دایره‌ای برای نسبت اجراهای موفق/ناموفق (اختیاری)
- دکمه حذف با دیالوگ تأیید «آیا از حذف پروژه مطمئنی؟»

## پرامپت ۴: صفحه ساخت پرامپت جدید

### هدف صفحه
نوشتن یک پرامپت جدید و انتخاب مدل‌های AI برای اجرا.

### APIهای مصرفی
- `GET /ai-models` — دریافت لیست مدل‌های فعال
- `POST /projects/{id}/prompts` — ایجاد پرامپت

### فیلدهای فرم
| فیلد | نوع | الزامی | محدودیت |
|------|------|--------|----------|
| متن پرامپت | Textarea (چند خط) | بله | ۱-۱۰۰۰۰ کاراکتر |
| مدل‌های AI | چک‌باکس لیست | خیر | یکی یا چندتا |

### المان‌های مورد نیاز
- **Textarea بزرگ** با شمارنده کاراکتر (مثلاً ۱۵۰/۱۰۰۰۰)
- **لیست مدل‌های AI** با چک‌باکس — هر آیتم: نام مدل + نام provider (مثلاً GPT-4o · OpenAI)
- **پیش‌نمایش سریع**: نمایش تعداد کاراکترها و تعداد مدل‌های انتخاب‌شده
- **دکمه «ایجاد پرامپت»** — غیرفعال اگر متن خالی باشه
- **دکمه «انصراف»** — برگشت به صفحه جزئیات پروژه

### وضعیت‌ها
- **بارگذاری مدل‌ها**: Skeleton لیست مدل‌ها
- **خالی (بدون مدل فعال)**: پیام «هیچ مدل AI فعالی یافت نشد. لطفاً با مدیریت تماس بگیرید.»
- **ارسال**: دکچه لودینگ
- **خطا**: پیام خطا بالای فرم
- **موفق**: هدایت به صفحه جزئیات پرامپت

### نکات UI/UX
- فونت monospace برای textarea پرامپت
- خط‌شمار (line numbers) کنار textarea (اختیاری)
- امکان بزرگ‌کردن textarea (resizable)
- دکمه «انتخاب همه» / «لغو انتخاب همه» برای مدل‌ها
- راست‌چین برای متن فارسی

## پرامپت ۵: صفحه جزئیات پرامپت و اجرا

### هدف صفحه
مشاهده جزئیات پرامپت، مدیریت مدل‌های متصل، اجرای پرامپت و دیدن نتایج.

### APIهای مصرفی
- `GET /projects/{id}/prompts/{pid}` — جزئیات پرامپت + مدل‌ها
- `POST /projects/{id}/prompts/{pid}/run` — اجرای پرامپت
- `GET /projects/{id}/prompts/{pid}/models` — مدل‌های متصل
- `DELETE /projects/{id}/prompts/{pid}` — آرشیو پرامپت
- `POST /projects/{id}/prompts/{pid}/models/{model_id}` — افزودن مدل
- `DELETE /projects/{id}/prompts/{pid}/models/{model_id}` — حذف مدل

### ساختار صفحه

**۱. هدر پرامپت:**
- تاریخ ایجاد
- دکمه‌های «اجرا», «آرشیو (حذف)»

**۲. متن پرامپت (بخش اصلی):**
- نمایش کامل متن با فونت monospace
- دکمه کپی متن

**۳. مدل‌های متصل:**
- لیست تگ‌های مدل‌های AI (نام + provider)
- دکمه «افزودن مدل» → باز شدن مودال انتخاب مدل

**۴. نتایج آخرین اجرا (بخش مهم):**
- جدول خلاصه با ستون‌ها: مدل AI, وضعیت (success/failed), تعداد برندهای پیدا شده, برندهای جدید, برندهای تکراری, پیام خطا (در صورت failure)
- اگر هنوز اجرا نشده: دکمه بزرگ «اجرای پرامپت»

**۵. صفحه اجرا (حالت لودینگ):**
- انیمیشن پیشرفت برای هر مدل
- نمایش مرحله به مرحله: "در حال ارسال به مدل..." → "در حال استخراج برندها..." → "پایان"
- بعد از اتمام: نمایش خودکار نتایج

### وضعیت‌ها
- **لودینگ**: Skeleton
- **قبل از اجرا**: دکمه Run پررنگ
- **حین اجرا**: Progress bar + وضعیت هر مدل
- **پس از اجرا**: نمایش نتایج
- **خطا**: پیام خطا + دکمه تلاش مجدد
- **آرشیو شده**: پیام «این پرامپت غیرفعال شده» + دکمه بازگردانی (فعلاً نداریم)

### نکات UI/UX
- اجرا زمان‌بره (چند ثانیه) — حتماً لودینگ باحال داشته باشه
- خروجی هر مدل جداگانه نمایش داده بشه
- دکمه آرشیو با دیالوگ تأیید
- لینک به صفحه رتبه‌بندی برندهای همین پرامپت

## پرامپت ۶: صفحه رتبه‌بندی برندها

### هدف صفحه
نمایش رتبه برندهای استخراج‌شده از یک پرامپت به صورت جدول و نمودار.

### APIهای مصرفی
- `GET /prompts/{id}/latest-rankings` — آخرین رتبه‌بندی‌ها (paginated)
- `GET /prompts/{id}/rankings` — همه رتبه‌بندی‌ها
- `GET /brands/{id}` — جزئیات یک برند
- `GET /brands/{id}/history` — تاریخچه رتبه یک برند در طول زمان

### ساختار صفحه

**۱. فیلترها:**
- انتخاب مدل AI (dropdown)
- انتخاب برند (dropdown)
- محدوده تاریخ (date picker)

**۲. جدول رتبه‌بندی:**
- ستون‌ها: رتبه, نام برند, دامنه, مدل AI, تاریخ
- مرتب‌سازی بر اساس رتبه (صعودی/نزولی)
- صفحه‌بندی
- کلیک روی نام برند → باز شدن مودال جزئیات برند

**۳. مودال جزئیات برند (یا صفحه جدا):**
- نام برند + دامنه
- آمار: تعداد ظهور, میانگین رتبه, بهترین رتبه, بدترین رتبه
- خط زمانی رتبه برند (نمودار ساده)

### وضعیت‌ها
- **لودینگ**: Skeleton جدول
- **خالی (بدون برند)**: «هنوز برندی استخراج نشده. اول یک پرامپت اجرا کن.»
- **خطا**: پیام خطا

### نکات UI/UX
- قابلیت دانلود خروجی به صورت CSV/Excel (اختیاری)
- نمودار خطی برای روند رتبه هر برند (اختیاری — با کتابخونه Chart.js یا مشابه)
- فیلترها در یک نوار بالای جدول
- نمایش رتبه با رنگ‌بندی (طلایی=۱, نقره‌ای=۲, برنزی=۳)
